# Kanban board

The **Kanban** section tracks the GitHub issues linked to your
[topics](/features/topics) on a board that spans your projects — a bird's-eye view
of work in flight, grouped by status.

<Screenshot src="/screenshots/kanban.png" alt="A Kanban board with columns of issue cards drawn from linked topics" caption="A Kanban board over the issues your topics are linked to, across projects." />

## What it shows

Because every topic can be linked to a GitHub issue, Precursor already knows the
state of your work. The board rolls those linked issues up into columns so you
can see, at a glance, what's open, in progress, and done across the projects your
topics touch.

Each card ties back to its topic, so the board doubles as a launcher: jump from a
card straight into the conversation that's driving that issue.

## Enabling the board

The Kanban section is optional and appears as a card on the home launcher and a
tab in the sidebar when enabled. Like every section, it's registered in the
sidebar, the home grid, and the **command palette** (⌘K / Ctrl-K) so you can jump
to it from anywhere.

::: tip Pairs with the scheduler
Combine the board with [scheduled topics](/features/scheduler) — e.g. a nightly
`/gh-sync` that refreshes issue state — to keep the board current without manual
polling.
:::
